增强现实肿瘤定位手术辅助系统

我们实验室正在开发一套用于手术过程中实时肿瘤定位与可视化的增强现实（AR）肿瘤成像系统。项目的核心目标是在医生进行肿瘤切除手术时，帮助医生实时识别肿瘤的位置和边界，从而提高切除精度并减少对正常组织的损伤。其基本原理是利用特定波长的近红外光或其他光学激发光源照射人体组织，由于肿瘤组织与正常组织在细胞结构、血管分布、代谢特征等方面存在差异，因此对光的吸收、散射、反射以及可能产生的荧光响应也会有所不同。系统通过捕获这些返回的光学信号，并对其进行图像处理和特征增强，从而识别出肿瘤区域。

整个系统从工程上主要分为四个部分。第一部分是激光照明系统，负责产生特定波长和稳定功率的激光，为组织提供光学激发；第二部分是光学成像与传感系统，负责接收组织返回的光学信号并完成数据采集；第三部分是图像处理与分析系统（Image Processing System），负责对采集到的数据进行校准、增强和目标识别，提取肿瘤区域信息；第四部分是增强现实显示系统，将识别出的肿瘤区域实时叠加到医生佩戴的AR设备视野中，使医生能够直接在手术过程中看到高亮显示的肿瘤位置和边界。最终目标是构建一套从激光激发、光学成像、数据处理到AR显示的完整术中导航系统，为肿瘤手术提供实时、直观的辅助决策能力。

面向模拟电路设计的大语言模型训练与仿真反馈优化系统

该项目的目标是训练一款具备模拟电路设计能力的领域专用大语言模型，使其能够根据自然语言描述的电路设计需求，自动生成对应的模拟电路拓扑结构，并通过仿真反馈不断修正和优化设计结果。传统模拟电路设计高度依赖硬件工程师和模拟电路工程师的专业经验，工程师通常需要根据设计指标手动构建电路拓扑、选择器件连接方式、调整器件尺寸参数，并反复通过 SPICE 仿真验证电路性能，例如增益、带宽、功耗、相位裕度、输出摆幅、稳定性等指标是否满足要求。这个过程专业门槛高、迭代周期长，并且很难完全自动化。因此，我们希望将大语言模型引入模拟电路设计流程，让模型不仅能够理解电路设计需求，还能够生成可仿真的 SPICE Netlist，并根据仿真结果进行自我修正，从而探索 LLM 在 Analog Circuit Design / IC Design 自动化中的应用能力。

围绕这个目标，我们首先为模拟电路设计任务建立了一套完整的任务定义和自动化评测流程。对于每一道电路设计任务，输入是文字形式的设计需求，例如要求设计某类放大器、电流镜、反相器、加法器或减法器，并给定对应的性能指标要求；模型需要输出对应的 SPICE Netlist，也就是电路拓扑结构。生成的 Netlist 会先经过自动化 sanity check，检查语法、器件连接、拓扑结构和基本可仿真性是否存在明显错误。如果拓扑结构通过初步检查，系统会进入 Bayesian Optimization sizing 阶段，在不同负载和设计条件下自动搜索器件尺寸参数，使当前拓扑尽可能达到最佳性能。随后，针对不同电路类型，我们构建了对应的 testbench，通过 PySpice / SPICE 仿真自动运行电路，并提取各项关键性能指标。最终，系统会根据仿真得到的指标与目标规格之间的差距，判断该电路是否满足设计要求，以及该设计在多大程度上是有效或优秀的。

在模型训练方面，整个项目并不是单纯做 benchmark，而是将这套自动化仿真验证流程作为后训练的核心基础。我们基于 Qwen 系列开源大语言模型进行领域化训练，数据来源包括开源电路设计数据，以及由实验室内部电路方向同学和老师共同构建的高质量模拟电路设计样本。训练过程主要围绕电路知识学习、拓扑生成能力和反馈修正能力展开：首先通过 SFT 让模型学习基础模拟电路知识、SPICE Netlist 表达方式以及不同电路类型的设计模式；随后针对“根据设计需求生成电路拓扑”的任务进行进一步监督微调，使模型能够从自然语言规格直接生成初始电路设计；在此基础上，我们将自动化仿真流程中产生的错误信息、sanity check 结果、sizing 结果和 testbench 仿真反馈重新组织成 feedback message，引导模型根据错误原因和性能差距对原始电路进行修改。这一部分使模型不仅能生成第一版电路，还能具备根据仿真反馈修复错误、调整结构和改进设计的能力。

进一步地，我们将仿真结果引入强化学习训练过程，把电路性能指标与目标设计要求之间的差距转化为 reward 信号。对于成功通过仿真的电路，系统会根据其增益、带宽、功耗、稳定性等指标与目标规格的匹配程度给予奖励；对于拓扑错误、仿真失败或性能严重不达标的设计，则根据失败类型和指标偏差给予惩罚或较低奖励。整个 RL 过程分别强化了模型的电路拓扑生成能力和基于反馈进行电路修正的能力。最终，项目形成了一套从自然语言需求输入、Netlist 生成、sanity check、BO sizing、testbench 仿真、指标评估、反馈生成，到 SFT 和 RL 训练优化的完整闭环，使模型逐步具备模拟电路自动设计和仿真反馈自我改进的能力。这个项目的核心价值在于，不只是让大语言模型“写出电路文本”，而是通过真实的仿真验证和性能指标反馈，将模拟电路设计任务转化为可评估、可训练、可优化的 AI for EDA 闭环系统。

SAFTAC: Simulation-Augmented Fine-Tuning of Open-Source LLMs for Analog Circuit Design
Junsheng Huang, Yifan Sun, Zhuoer Zhang, Ning Wei, Yening Liu, Michael Molter, Pavan Kumar Hanumolu, Elyse Rosenbaum, Bin Hu, Huan Zhang 
NeurIPS 2026 Conference Submission

TL;DR: We introduce a simulation-grounded dataset and a fine-tuning framework for analog circuit design, enabling open-source LLMs to generate and revise specification-conditioned circuit designs using SPICE-based feedback.

Abstract:
Circuit design for analog integrated circuits (ICs) is challenging because circuit topology, device sizing, and circuit performance are tightly coupled. Existing large language model (LLM)-based methods either rely on inference-time correction around frozen models or train on limited circuit types without directly using simulation outcomes as training signals. A key bottleneck is that existing datasets do not provide sufficient task information for training LLMs on end-to-end specification-conditioned design. To fill this gap, we first construct a large-scale simulation-grounded dataset with 8,626 design tasks across 10 different types of analog circuits, where each task includes a textual description, labeled I/O ports, loading condition, target specifications, a testbench, and a reference netlist. Using this dataset, we present SAFTAC, a simulation-augmented fine-tuning framework that trains LLMs to generate and revise analog circuit design. SAFTAC combines three-stage supervised fine-tuning, which progressively builds the capabilities required for analog circuit design and feedback-guided correction, with simulation feedback-augmented reinforcement fine-tuning, which further optimizes models using simulation-grounded rewards. We evaluate SAFTAC under single-pass generation and two-pass generation with simulation feedback, showing improvements over the strongest closed-source LLM by 15.9% and 7.8%, respectively.

教育大模型后训练与数学解题可视化生成能力探索

这一段实习项目面向在线教育与搜题软件场景，核心目标是将传统“拍照识题—题库检索—匹配已有答案”的搜题模式，升级为由大语言模型直接完成题目理解、推理求解、步骤讲解和教学辅助的 AI 解题系统。团队基于开源通用大语言模型搭建了教育领域后训练体系，先利用大规模教育语料对基座模型进行继续预训练，使模型具备更强的教育领域知识表达和题目理解能力，再通过完整的 SFT pipeline 对具体任务能力进行快速训练和验证。在这一基础上，我负责探索数学解题过程中的可视化生成能力，即通过构建专项 SFT 数据和配套渲染流程，使模型在生成文字解析的同时，能够根据题目内容自动生成辅助理解的函数图、几何图、数轴图、线段图或应用题示意图，从而让 AI 解题结果从纯文字答案升级为图文结合的教学化讲解。

该任务的技术难点在于，数学辅助图像不是普通的开放式图片生成，而是需要严格表达题目中的数学关系、坐标位置、函数形状、几何结构和标注信息，因此对准确性、可控性和可复现性要求很高。项目中曾考虑过基于 Diffusion 的图像生成路线，但这类方法更适合生成自然图像或风格化插图，在数学教育场景中存在明显限制：一方面，Diffusion 生成结果难以保证坐标、比例、角度、函数曲线和几何约束的精确正确；另一方面，其输出难以被规则系统检查、修改和稳定复现，也不利于和现有解题流程中的结构化内容对齐。因此，该项目最终重点采用“LLM 结构化输出 + 程序化渲染”的路线：不直接让模型生成最终图片，而是让模型生成标准化 JSON 字段、自定义绘图指令或受约束的绘图代码，再由外部规则脚本解析字段，并调用 Python 绘图库生成最终图像。

围绕这一技术路线，我完成了从任务设计到实验验证的完整探索。具体包括：定义数学解题可视化任务的输入输出格式，筛选函数、几何、数轴、线段和应用题等适合插图的题型；在原有数学题与标准解析基础上构建 SFT 样本，补充图像插入位置、图像类型、关键数学元素、结构化绘图字段和对应渲染逻辑；设计模型输出规范，使模型能够在解题过程中判断是否需要图像，并生成可被后端系统稳定解析的绘图内容；同时实现配套的后处理与渲染流程，将模型输出的结构化字段转化为具体绘图逻辑，生成辅助学生理解的数学图像。相比直接生成 Python 代码的方式，结构化字段方案降低了模型输出复杂度，减少了语法错误和不可控绘图结果，使图像生成过程更稳定、可验证，也更适合真实教育产品场景中的部署需求。

整体来看，这段工作依托团队已有的教育大模型继续预训练与 SFT 后训练平台，围绕“数学解题模型如何具备可视化讲解能力”这一新任务方向，完成了任务定义、技术路线选择、SFT 数据构建、结构化输出设计、渲染系统实现与模型效果验证。其核心价值在于把大语言模型的文字推理能力与程序化图像生成能力结合起来，使 AI 解题系统不仅能够给出答案和步骤，还能够通过可控、准确、可复现的辅助图像提升学生对数学推理过程的理解。

在爱奇艺机器学习工程实习期间，我参与了面向影视内容生产场景的 AI 视频生成能力探索。爱奇艺作为在线视频与影视内容平台，同时涉及剧集、电影和短视频内容生产，因此希望评估生成式 AI 在影视制作流程中的应用潜力，例如辅助生成特定人物镜头、降低部分实拍成本、提升素材生产效率等。传统 Text-to-Video 模型虽然能够根据文本描述生成视频，但在影视业务中，仅依靠文本提示很难精确控制画面中人物的姿态、动作节奏、镜头连续性和动态一致性，难以满足真实拍摄场景中对人物动作和镜头表达的强控制需求。因此，该项目的目标是在开源文生视频框架基础上，引入额外的结构化视觉控制信号，使模型能够在文本描述之外，根据骨骼姿态序列控制人物动作生成。

技术上，项目基于开源视频生成框架 OpenSora / Open-Sora 进行复现与实验，整体模型路线属于 Diffusion-based Text-to-Video，核心生成结构采用 Diffusion Transformer（DiT）类架构，通过扩散模型逐步去噪生成视频内容，并利用 Transformer 建模视频中的空间与时间特征。为了增强生成过程的可控性，项目进一步参考 ControlNet 的条件控制思想，将人体骨骼姿态作为额外控制输入接入视频生成 pipeline。ControlNet 的核心作用是在预训练扩散模型基础上引入额外条件分支，使模型在保留原有文本生成能力的同时，能够受边缘、深度、人体姿态等结构信号约束；在本项目中，对应的控制信号主要是从真实视频中提取并重新渲染得到的骨骼姿态序列，用于约束生成视频中人物的动作形态和时序变化。

围绕这一目标，我参与了从模型调研、数据构建到训练 pipeline 改造的完整流程。前期调研了 Text-to-Video、Video Diffusion、Diffusion Transformer、Pose-Controlled Generation 和 ControlNet 等相关方法，分析不同视频生成模型在画面质量、时序一致性和条件控制能力上的差异。数据侧则基于公司内部影视素材库，筛选和切分包含人物动作的视频片段，并使用计算机视觉工具和关键点检测方法提取视频中人物的身体、面部、眼部、嘴部等关键点信息；随后通过 Python 脚本将逐帧关键点重新合成为与原视频时间轴对齐的骨骼动作视频，构建“文本描述 / 原始视频 / 姿态控制视频”一一对应的训练样本。模型侧则在 OpenSora 的训练和生成流程基础上，尝试将 ControlNet 式条件控制模块接入 DiT 视频生成结构，使模型能够同时接收文本 prompt 和骨骼姿态视频作为输入，并学习根据姿态序列生成动作一致的人物视频。

整体来看，项目不仅关注模型能否根据文本生成视频，更关注其是否能够满足影视场景中对人物动作、姿态对齐和动态一致性的控制要求。通过构建骨骼姿态控制数据、改造视频生成训练 pipeline，并将 ControlNet 条件控制思想融入 OpenSora / DiT 视频生成框架，项目为后续在业务镜头生成、人物动作控制和 AI 辅助影视制作中的应用提供了前期技术验证。
